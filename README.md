# 株式会社LIFEFUND PR実績アーカイブ

> プレスリリース・外部メディア掲載・講演登壇・受賞歴・PRトピックを蓄積する**公開実績アーカイブ**。  
> **Googleスプレッドシートを更新するだけでサイトに自動反映されます。**

---

## ⚠️ 重要：このサイトに載せてよい情報のルール

このアーカイブは**外部公開**されており、メディア関係者の取材リソースとして使われます。
以下のルールを必ず守ってください。

### ✅ 載せてよいもの（公開済み情報のみ）

- PR TIMES等で**配信済み**のプレスリリース
- すでに**公開・放送・掲載された**メディア露出
- すでに**開催済み**の講演・登壇（または公式に告知済みの予定）
- **正式に発表された**受賞歴
- **公開可能な**PR上の取り組み・事業情報

### ❌ 載せてはいけないもの

- **未公開のリリース**（配信予定日が未到来のもの）
- **下書き・社内検討中**の情報
- **社内メモ・個人情報**
- 取引先・顧客の未公開情報
- **未確定の交渉・契約情報**

### 📌 未来目標・ビジョンの扱い

「BLUEPRINT 2036」「中期経営計画」「将来の展望」などの**未来目標**は、
過去の**実績と混在しないよう**専用の `vision` タブで管理してください。
サイト上も「未来目標であり実績ではない」旨が明示的に表示されます。

---

## 📝 運用フロー（PR担当の方へ）

1. **情報が公開される**（PR TIMES配信完了 / 記事公開 / イベント終了 など）
2. スプレッドシートの該当タブに行を追加
3. **`meta` タブの `last_updated` を更新**（運用ルール）
4. 数分以内にサイトに自動反映

---

## 🚀 初期セットアップ（最初の1回のみ）

### STEP 1. Googleスプレッドシートを準備

以下の**7タブ**を作成します。

| # | タブ名 | 用途 |
|---|---|---|
| 1 | press_releases | プレスリリース（公開済み） |
| 2 | media_coverage | 外部メディア掲載（公開済み） |
| 3 | speaking | 講演・登壇（開催済み or 公式告知済み） |
| 4 | awards | 受賞歴（正式発表済み） |
| 5 | vision | **未来目標・ビジョン**（実績と分離） |
| 6 | topics | PRトピック・事業説明 |
| 7 | meta | **データ最終更新日など** |

### STEP 2. スプレッドシートを「ウェブに公開」

1. ファイル → 共有 → ウェブに公開 → 公開
2. 右上「共有」→「リンクを知っている全員」「閲覧者」に変更

### STEP 3. ID と GID を `config.js` に貼り付け

各タブを開いた時のURL末尾 `gid=◯◯◯` の数字をメモして貼り付けます。

### STEP 4. GitHub Pages で公開

Settings → Pages → main / root を選択 → Save

## 📋 スプレッドシートのテンプレート

### タブ1: `press_releases`（公開済みのみ）

| date | title | url |
|------|-------|-----|

- **date**: PR TIMES等で**配信された日**（`YYYY-MM-DD`）
- **未配信のものは絶対に入れない**

### タブ2: `media_coverage`（公開済みのみ）

| date | title | brand | url1 | label1 | url2 | label2 |
|------|-------|-------|------|--------|------|--------|

- **date**: 記事や番組が**公開・放送された日**
- **取材を受けただけで未公開のものは入れない**

### タブ3: `speaking`（開催済み or 公式告知済み）

| date | event | venue | session | theme | attendees | satisfaction | event_url | video_url | report_url |
|------|-------|-------|---------|-------|-----------|--------------|-----------|-----------|------------|

- 開催前のものは、**主催側が公式に告知済み**の場合のみ掲載可

### タブ4: `awards`（正式発表済み）

| date | title | detail | url | crown |
|------|-------|--------|-----|-------|

- 公式発表前の受賞情報は入れない
- 殿堂入りは `crown` に `true`

### タブ5: `vision`（未来目標・ビジョン専用）🆕

| target_year | name | detail | url | url_label |
|-------------|------|--------|-----|-----------|
| 2036 | BLUEPRINT 2036（10年ビジョン） | 2026年1月発表。2036年までに売上150億円... | | |

- **このタブには未来目標・将来計画のみ**を入れる
- 実績は他タブへ（混在禁止）

### タブ6: `topics`（PRトピック・事業説明）

| topic | detail | url | url_label |
|-------|--------|-----|-----------|

- 進行中の事業説明、ブランド紹介などに使用
- **未来計画は `vision` タブへ**

### タブ7: `meta`（データ最終更新日）🆕

縦持ち形式：

| key | value |
|-----|-------|
| last_updated | 2026-04-14 |

- **ルール：アーカイブを更新したら必ず `last_updated` も更新する**

## 🛠 ファイル構成

```
.
├── index.html         ← 本体
├── app.js             ← 動的レンダリング処理
├── config.js          ← スプレッドシートID設定（初回のみ編集）
├── fallback-data.js   ← スプレッドシート未設定時のフォールバック
└── README.md          ← このファイル
```

## ❓ トラブルシュート

- **「データ取得に失敗しました」**  
  → スプレッドシートの公開設定と共有設定を確認

- **新しい行を追加したのに反映されない**  
  → スーパーリロード（Ctrl+Shift+R / Cmd+Shift+R）

- **「データ最終更新日」が変わらない**  
  → `meta` タブの `last_updated` 行を更新したか確認
